What you see here is a work-in-progress version of a new “molecules view” for the Lab project. The nice feature of this view is that it behaves almost exactly like an HTML <img> element and doesn't “pollute” the CSS stylesheet of the document that contains it (in particular, it doesn't modify the em size, and therefore the text rendering, of its containing document.)

What that means for developers and users of the Lab project is that they can adjust the size and behavior using the same CSS (or Javascript) techniques that you would use for any other elements on a page. In particular, it will be possible to apply standard responsive design techniques to Lab pages without having to make special exceptions for molecule views. This will greatly help traditional web designers work effectively with the project.

This particular molecules view scales smoothly with the width of the page. However, that behavior is determined entirely by the CSS applied to it. You could have a fixed width, with the height being determined by the model contents. On the other hand, if you wanted to make sure the molecule view always fit in the page height, you could leave the width unspecified and use a height media query, or a one-line jQuery window-resize handler, to set the height of the DOM element containing the molecule view—no special communication with or knowledge of the Lab molecules view is required.
