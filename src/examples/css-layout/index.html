What you see here is a work-in-progress version of a new “molecules view” for the Lab project. The nice feature of this view is that, for the web designer, it behaves almost exactly like a familar HTML <img> element with a few extra properties that can be customized using pure CSS. For developers, this means that page layout concerns can be handled the same way for molecule views as it is for other elements of an activity or interactive.

In particular, it will be possible to apply standard responsive design techniques to Lab pages without having to make special exceptions for molecule views. This will greatly help traditional web designers work effectively with the project. For example, the axis labels and title of the graph on this page were customized for legibility and attractiveness at different window sizes using standard CSS techniques, and would be further tweakable by a web designer. Developers can therefore concern themselves with semantics and leave layout details to designers (or leave layout details for the times when they put on their “designer” cap.)

In this example, CSS media queries are used to adjust the width of the molecules view when the window width changes, with the browser automatically maintaining the height of the view to maintain the aspect ratio that was determined when the model initially loaded. (This is analagous to the way images are handled in HTML.)

If you wanted to make sure the molecule view always fit in the page height, you could leave the width unspecified and use a height media query, or a one-line jQuery window-resize handler, to set the height of the DOM element containing the molecule view—no special communication with or knowledge of the Lab molecules view is required.
