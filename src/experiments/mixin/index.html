JavaScript Mixin Experiments

Inspired by Angus Croll's article A Fresh Look at JavaScript Mixins (particularly section 4: Adding Caching) I put together this sample of mixin patterns.

The last example I created shows a pattern I hadn't seen before. The mixin addMixin4 adds a counter() function that creates it's own closure variable in the scope of the function in the Model object it is mixed into and also has a closure variable that is scoped to the mixin itself. This second closure variable is changed by any Model object use of the counter() function.

This is experimention done while thinking about extracting the code we use in the MD2D model that manages getting and setting properties and managing property change listeners and events.
